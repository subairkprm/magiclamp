# MagicLamp PR Handoff Template

> Use this in every PR body. All sections are mandatory.

## Claim
What change is being made and why for this single issue/workstream.

## Evidence
Concrete proof (tests, screenshots, logs, docs references) supporting the claim.

## Files changed
Enumerated file list with short rationale per file.

## Validation performed
Commands/checks run and outcomes.

## Security impact
Describe auth, permissions, secrets, attack surface effects (or explicit none).

## UAE data residency impact
Explain if any data path, processor, or storage boundary changes. Confirm UAE residency is preserved or detail risk.

## Remaining risk
Known limitations, assumptions, and unresolved concerns.

## Founder action needed
Explicit decision requested from founder (approve/reject/follow-up).

## Next recommended workstream
Single next issue/workstream recommendation; do not mix into current PR.

## Routing metadata
- Risk band: RED / AMBER / GREEN
- Hard-gate triggered: Yes / No
- Gate reason(s):
